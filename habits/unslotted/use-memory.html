use-memory — Habit

Why

a turn would benefit from what has already been learned — a new task area, an ambiguous instruction, a recurring lens or collaborator — or a durable, reusable lesson crystallizes that a future turn should not have to re-derive

Action

Reaching back: call recall and read what returns before acting — liberally, since it is cheap and pre-cached on nearly every turn.

Saving forward: when a durable lesson crystallizes, record it in both memory systems — the rich version in your native memory, and a small learn call in the shared KCD store, at the appetite your active write-memory pole sets.

Explanation

Two memory systems run side by side, and a durable lesson earns a place in each: the native store holds the rich, per-agent version; the KCD store holds a tiny, shared claim any agent under the lens can recall. They are kept small on purpose — many little memories beat a few big ones — so writing to both is cheap. Recall liberally: deliberate retrieval is what keeps a memory alive, and a preloaded lesson earns back its context cost many times over. Argument shapes, caps, and the tag vocabulary live with the recall, learn, and known_tags tools — not here. The full model is in the memory-system plan (retired, Phases 1-3 delivered).
