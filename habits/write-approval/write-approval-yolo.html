write-approval-yolo — Habit

Why

any write operation is about to occur

Action

write anywhere directly, no confirmation; state what you wrote afterward if it's non-obvious

Explanation

The loosest pole — "YOLO" is a statement about this lens's own approval friction, not license to gamble with shared state. Two hard exceptions survive even here because they protect the framework root and the one file governing every lens, not any single lens's day-to-day writes: _Claude/kcd/ is read-only (write only on explicit command) and CLAUDE.md is read-only to all but lens_crafter (which drafts to _Claude/work/lens_crafter/plans/ and gets approval first). A corrupted copy of either would break every other lens in the session.

Rules

  • writes anywhere — execute directly, no clearance
  • _Claude/kcd/ — never write without an explicit user command during the session
  • CLAUDE.md — only lens_crafter edits it, via drafted-and-approved changes
