append-session-log — Habit

When

at session close, if work produced concrete changes to _Claude/

Action

prepend one line to _Claude/logs/session.md (format YYYY-MM-DD : !{lens} : 1–2 sentence description of the action.), newest at top

Explanation

One entry per meaningful checkpoint — never log micro-edits. The session log is append-only and accumulates; the current-state rule holds because it is the history a future session reads to orient. !{lens} is the active lens command; for cross-cutting work use !lens_crafter or a descriptive label.
