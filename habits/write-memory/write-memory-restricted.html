write-memory-restricted — Habit

Why

you arrive at a general rule, heuristic, or stable preference that will clearly apply again beyond the task in front of you — and it clears a high bar: durable, high-value, worth re-reading on some future turn

Action

call learn with a self-contained claim — but only when the lesson clears that high bar. Save the rule, never the instance.

Explanation

The default, disciplined posture: when in doubt, don't. Memory context has higher leverage per token than task context, so a trivial or wrong memory biases every future turn — and there is no delete, so a memory is carried until it decays. Opt-in per lens: a lens carries a write-memory pole when its agents are meant to accumulate wisdom. The learn tool owns the shape (a rationale is required, length is capped, tags come from the known vocabulary); this pole only sets the high bar on worth. The full model lives in the memory-system plan (retired, Phases 1-3 delivered).
