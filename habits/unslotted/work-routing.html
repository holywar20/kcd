work-routing — Habit

Why

about to write any session output (inherited by every lens via _lens_base)

Action

write every artifact under the loaded lens's working space — AI output to _Claude/work/{lens}/AI/, the user's docs to _Claude/work/{lens}/human/, plans to _Claude/work/{lens}/plans/ — always the full path from root, never a bare AI/

Explanation

This space is scratch — temporary artifacts before promotion; default to writing things down here rather than holding them in the conversation, because a written artifact is faster to align on than paragraphs of chat and costs nothing to discard. Plan liberally (a plan needn't be earned), experiment and noodle without asking first, research live (read the actual code, run the actual query), and stay disposable — scratch needs no polish. Promotion from work/ to a permanent home (references/, plans/, committed code) is always a separate, explicit, later step the user decides; the agent never promotes its own output by writing something "nice" the first time. The routing and liberal-scratch posture are project defaults a deployed copy may adjust.

Rules

  • write the whole path from root every time — never a bare AI/ trusting the working directory to place it
  • never write work output directly to references/ or lenses/ (read-only during sessions), or to another lens's work folder
  • when more than one lens is loaded, {lens} is the primary (first-loaded) lens — it owns the work, cross-cutting plans included; there is no work/cross/
  • promotion from work/ to a permanent location is always a separate, explicit, later step
