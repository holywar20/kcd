habits — Nav index

Atomic single-step reflexes. Deployed and active in this instance. Folders reflect control tier, not topic: a habit-class mutex group gets its own folder named after the class, and every habit with no class lives under unslotted/. Folder placement is purely organizational — a habit's class comes from its own habit-class frontmatter field, never from where the file sits on disk.

Tiered — habit-class mutual-exclusion slots

Each row below is one pole of a class; a lens's Habits table wires in exactly one pole per class via data-kcd-habit-class on the slot. * marks the pole every lens gets today via _lens_base unless it swaps in another pole from the same row group.
