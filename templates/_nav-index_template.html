Nav-index (template)

Scaffold note

A nav-index is a folder's table of contents — one row per artifact, in the What | Where | Why shape (the index-format habit). It is how an agent finds the candidate files in a folder by name and use-case without reading each one. Every folder that holds artifacts carries exactly one, named nav-index.html.

Copy to {folder}/nav-index.html (always that basename — one per folder). Fill every {placeholder} and delete these scaffold notes. The authored nav-index's frontmatter replaces this template's identity fields:

  • type → nav-index
  • status → active — a nav-index is live in place; it ships and reads directly, it is never a deployable seed
  • updated → the run date

Group the rows into multiple data-kcd-sections when the folder has natural clusters (give each an <h3>); a small folder uses a single section. The row order and grouping are human-facing; only the What | Where | Why cells are addressed.
