KCD Primitives — SDK Reference

Why

working with the KCD SDK's object model — artifact types, roles, or the wire shape

Concrete TypeScript classes in kcd_sdk/src/primitives/ that parse, validate, and serialize every KCD artifact type. The conceptual side lives in kcd_framework; this document covers the SDK implementation — types, validation rules, roles, and the parse pipeline.

⚠️ Staleness note — parse mechanics moved to HTML

This reference predates the Phase-2 HTML substrate cutover. The parse and serialize mechanics it originally documented — the markdown pipeline (splitFrontmatter → parseBody (## -split) → extractLinks), runInit, the factory registry, toMarkdown, and command-style frontmatter — are gone: HTML is now the sole substrate, entry is fromHtml (validate-first via KcdValidate / KcdParse), and writing awaits the Phase-3 emitter. The type/role/wire tables below remain the durable object-model reference; for the current substrate see kcd-document-protocol. pipeline has been dropped from the type union and its class deleted — the table below reflects the current union. The index→nav-index rename is still a deferred follow-up. A full rewrite of this reference is pending.

Type System

ArtifactType

Every artifact carries one of these type discriminators (declared in frontmatter as type:):

ValueClassDetected by

planPlanObjectfrontmatter type: plan + plans/ directory
referenceReferenceObjectfrontmatter type: reference + references/ directory
generatorGeneratorObjectfrontmatter type: generator + generators/ directory
analyzerAnalyzerObjectfrontmatter type: analyzer + analyzers/ directory
habitHabitObjectfrontmatter type: habit + habits/ directory
contractContractObjectfrontmatter type: contract + contracts/ directory
templateTemplateObjectdirectory convention (kcd/templates/); frontmatter check skipped
frameworkFrameworkObjectdirectory convention (kcd/, not kcd/templates/); frontmatter check skipped
unknownKCDPrimitive (base)fallback; nav-index files (nav-index.html) and unclassified paths

LensObject.classifyByPath() performs the authoritative path-to-type mapping at scan time.

KCDRole

Determines which context dock an artifact belongs to in a host UI:

RoleArtifacts
'know'LensObject returns 'lens' (composite); PlanObject, ReferenceObject, FrameworkObject, TemplateObject return 'know' (default)
'do'HabitObject, ContractObject, GeneratorObject, AnalyzerObject, UtilityObject
'lens'LensObject only

Base KCDPrimitive.getRole() returns 'know'; do-role classes override it explicitly.

Parse Pipeline

Every artifact runs the same init pipeline on construction:

splitFrontmatter() → validateFrontmatter() → parseBody() → validateStructure() → extractLinks()
  • splitFrontmatter — splits ----delimited YAML from body; throws KCDParseError on malformed YAML
  • validateFrontmatter — checks required frontmatter fields; throws KCDValidationError on failure
  • parseBody — splits body by ## headings into sections: Record<string, string>
  • validateStructure — checks required sections; throws KCDValidationError on failure
  • extractLinks — walks all [text](href) in body; classifies as internal | external | anchor; records which ## section each link was found in

Two init paths exist:

  • runInit(markdown) — parses raw markdown; full pipeline
  • runInitFromScanned(ScannedFile) — trusts pre-split frontmatter from the scanner; skips splitFrontmatter

Base Class — KCDPrimitive

kcd_sdk/src/primitives/framework/KCDPrimitive.ts

The shared parse chain, factory registry, and serialization layer. All concrete classes extend it.

Factory registry — KCDPrimitive.register(type, fn) maps each ArtifactType to a factory. KCDPrimitive.create(type, markdown, absPath) dispatches through the registry; falls back to KCDPrimitive.createBase() for unregistered types. All concrete subclasses self-register at module load time.

Key static methods:

MethodPurpose
KCDPrimitive.create(type, md, path)Dispatch through factory registry
KCDPrimitive.parse(md, path)Parse as unknown/base type
KCDPrimitive.fromScanned(ScannedFile)Hydrate from scanner output
KCDPrimitive.fromSerialized(SerializedArtifact)Hydrate from wire JSON; bypasses parse pipeline
KCDPrimitive.collectWrites(objects[])Collect dirty objects into a WriteMap

Key instance methods:

MethodPurpose
getRole()Returns KCDRole; 'know' by default
getPolicy()Returns PolicyEntry[]; [] by default (only LensObject produces policy)
typeCheck()Non-throwing validation; returns TypeCheckIssue[]
toContextBlock()Renders artifact as a context-window block

serialize()Returns SerializedArtifact for wire transfer

DREDGE_MAX = 4 — the maximum allowed dredge depth, clamped in clampDepth().

Framework-Layer Classes

LensObject

kcd_sdk/src/primitives/framework/LensObject.ts

The spine: owns project root, reads files, orchestrates dredge, assembles context.

Frontmatter required: type: lens, command (the !name invocation string)

Sections required: ## Know, ## Care, ## Do

Role: 'lens'

Dredge: parses the ## Know table for PolicyEntry rows. Rows whose why cell starts with always (case-insensitive) are auto-dredged at load time up to dredgeDepth (default 2, clamped to DREDGE_MAX = 4). Non-always entries appear in the ## Available on request stub block appended by serializeForContext().

Key extras:

Method/FieldPurpose
LensObject.load(path, opts?)Full load: parse + dredge; primary entry point
LensObject.inferProjectRoot(path)Walks ancestors until _Claude/ found
LensObject.classifyByPath(abs, root)Path → ArtifactType; used by scanner and dredge
LensObject.resolveHref(href, root)Vault-root-relative href → absolute path
serializeForContext()Assembles dredged nodes into one context blob
getPolicy()Returns parsed PolicyEntry[] from Know table
getNodes()Returns dredged KCDPrimitive[]

readFile() is an overridable seam — the MCP server can swap it out for a custom reader without subclassing. The default reads via fs.readFileSync.

PlanObject

kcd_sdk/src/primitives/framework/PlanObject.ts

A Know artifact: digest of ongoing work and intent. Loaded for context, not execution.

Frontmatter required: type: plan

Sections required: ## Goal, ## Approach, ## Phases

Role: 'know' (inherited)

ReferenceObject

kcd_sdk/src/primitives/framework/ReferenceObject.ts

Free-form prose loaded on demand. No required section structure.

Frontmatter required: type: reference

Role: 'know' (inherited)

FrameworkObject

kcd_sdk/src/primitives/framework/FrameworkObject.ts

Conceptual foundations and conventions for the KCD system. Identified by directory (kcd/ but not kcd/templates/); frontmatter type check is skipped. No required sections.

Role: 'know' (inherited)

TemplateObject

kcd_sdk/src/primitives/framework/TemplateObject.ts

Scaffolds for authoring a specific artifact type. Identified by kcd/templates/ directory; frontmatter type check is skipped. No required sections.

Role: 'know' (inherited)

Procedure-Layer Classes

All procedure classes override getRole() to return 'do'.

HabitObject

kcd_sdk/src/primitives/procedure/HabitObject.ts

Atomic behavior fragment — leaf node, no dredge policy. Canonical four-field shape, in order: why · action · explanation · rules (rules optional; a don't-style habit omits action). The body is no longer free-form — those four fields project by code into a dense two-line agent form (a terse deck pointer when linked, the full two-liner when suggested), so why/action stay one terse clause each and depth lives in explanation. why was when until 2026-07-13 — renamed to match the canonical What|Where|Why convention; it's the same trigger prose a lens's Why cell can defer to via mode:habit. See _habit-template.

Frontmatter required: type: habit

Sections required: ## Why (+ ## Action for a do-style habit)

Role: 'do'

ContractObject

kcd_sdk/src/primitives/procedure/ContractObject.ts

Behavioral agreement between a lens and an agent. Must declare the trigger condition.

Frontmatter required: type: contract

Sections required: ## When

Role: 'do'

GeneratorObject

kcd_sdk/src/primitives/procedure/GeneratorObject.ts

Mechanical, manifest-driven builder. Must declare the execution procedure.

Frontmatter required: type: generator

Sections required: ## Do

Role: 'do'

AnalyzerObject

kcd_sdk/src/primitives/procedure/AnalyzerObject.ts

Judgment pass that reads broadly and writes one report. Must declare the execution procedure.

Frontmatter required: type: analyzer

Sections required: ## Do

Role: 'do'

Error Types
