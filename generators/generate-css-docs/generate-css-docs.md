# generate-css-docs — Generator (canonical)

*Generic across any project with a global CSS utility-class system. The deployed copy solves the
requirements below with project-specific values. Interpretive doc generator — sonnet, not haiku.*

Base rules: [_generator_base](_Claude/kcd/generators/_generator_base.md) — composition model,
requirement resolution, fail behavior, output conventions, reference wiring.

---

## Parameters

Output: the `reference-target` requirement (solved in the deployed copy)

*`name` / `#{name}` derive from the filename; `model` and `base` live in frontmatter.*

---

## Requirements

*Declared generically here; solved in the deployed copy. See `_generator_base` for pre-flight
resolution and the fail rule.*

| Name | Kind | Description |
|---|---|---|
| `style-source` | Input | Glob of the CSS files that define the global utility-class system. |
| `class-prefix` | Input | The prefix that marks a class as global (e.g. `g-`). |
| `reference-target` | Input | The reference file this generator writes — flush-and-fill. |
| `reference-lenses` | Input | The lens(es) whose Know table must point to the generated reference. |

---

## Care

**Defends:**
- The reference is flush-and-fill — fully regenerated each run, never hand-patched.
- Documents only classes carrying `class-prefix` — scoped and component CSS is out of scope.
- After a run, every lens in `reference-lenses` carries a correct Know-table entry for the reference.

**Will not touch:**
- The source CSS files — read-only.
- Component or scoped CSS, or any class without `class-prefix`.
- Lens content other than the single Know-table row for this reference.

**Surfaces (notes inline, does not block):**
- A prefixed class whose intent cannot be inferred from its CSS — document the raw declaration and flag "purpose unclear — verify".

---

## Do

*Pre-flight runs first, per `_generator_base` — deployment check, then requirement resolution.
Phases below assume the generator is deployed and all requirements are solved.*

### Phase 1 — Source Load

Read every file in `style-source`. Collect every class whose name begins with `class-prefix`,
with its full CSS declaration and the file it came from.

### Phase 2 — Reference Generation

For each global class, produce an entry:
- **Does** — one line: what the class is for.
- **CSS** — the declaration it applies.
- **Use when** — the situation that calls for it.
- **Expects** — the HTML structure it assumes.
- **Composes with** — related global classes commonly used alongside it.

Group entries by source file. Write a short header naming the source glob and the generation date.

### Phase 3 — Write Reference

Flush-and-fill `reference-target` with the generated reference.

### Phase 4 — Lens Wiring

For each lens in `reference-lenses`, ensure its Know table (References section) has a row
pointing to `reference-target`: `What | Where | Why`. Add the row if missing; correct the path
or Why text if stale. Touch nothing else in the lens.

*Skipped under `--test`, per `_generator_base` — a test run produces only the test artifact.*

### Phase 5 — Output Declaration

This generator writes `reference-target` (flush-and-fill) and edits the Know table of each lens
in `reference-lenses`. No other output.

**Reference format** — follows the reference template
([`_reference_template`](_Claude/kcd/templates/_reference_template.md)): frontmatter (with
`updated` set to the run date), then the body.

```markdown
---
type: reference
status: active
name: {kebab-case reference name}
description: "One line: the global CSS class system this documents"
updated: {run date}
---

# {Title}

*Generated by #generate-css-docs from {style-source}. Documents the `{class-prefix}` global
utility-class system. Flush-and-fill — do not hand-edit; rerun the generator.*

## {Group — e.g. Layout (layout.css)}

### `{class}`
- **Does:** {one line}
- **CSS:** `{declaration}`
- **Use when:** {when to reach for it}
- **Expects:** {HTML structure}
- **Composes with:** {related global classes}
```
