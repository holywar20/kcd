Root Context — Seed Source

Why

changing what an agent is told before it has read anything else, or adding support for a new agent host

Every agent host reads a differently-named file at the project root before it does anything else — CLAUDE.md, AGENTS.md, GEMINI.md. Those files are owned by conventions outside KCD (protocol §9), so KCD does not reformat them. It authors them instead: this document carries one §10 seed per host, and a deploy extracts the one that matches and writes it into a managed block at the top of that host's file.

This artifact is never copied. Only its payloads travel, and only into the file each payload names. Everything below a target's <!-- kcd:end --> marker belongs to the project and is never touched.

The payloads are deliberately thin. An entry file's whole job is to get an agent to the real context in one step and then get out of the way. Anything that is not "here is how you reach the vault" belongs in the entry document, not here — the entry file is a pointer, and a pointer that grows into a manual has failed at being a pointer.
