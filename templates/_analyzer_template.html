Analyzer (template)

Scaffold note

Scaffold for authoring a canonical analyzer. Copy to _Claude/kcd/analyzers/{name}/{name}.html (an analyzer is a folder: the .html trunk plus a context/ subfolder). Holds all generic interpretation logic; declares — never solves — its requirements, and names no project-specific path or value. Fill every {placeholder} and delete these scaffold notes.

The authored analyzer's frontmatter replaces this template's identity fields:

  • type → analyzer
  • status → active or disabled
  • lens → {lens} — the lens this analyzer composes with (gives it judgment; lets it run cold)
  • base → _analyzer-base

(model is no longer frontmatter — it is the model parameter below.)

Base rules: _analyzer_base — role, composition model, requirement resolution, output conventions.

Each parameter is a name / type / default triple — the typed-field vocabulary the app's setting fields use. model is one such parameter (the default model that seeds the work-graph node; not yet implemented); lens lives in frontmatter. name is the folder/file name and is not retyped here. An analyzer's write authority is report-only — it writes reports (typically one; a small number is fine), never source or canonical paths.

Know

An analyzer reads anywhere — source, references, and generator output (audits/). Declare the inputs this analyzer reads; audits are one input among many, not a precondition.

Phase 1 — Read

Phase 2 — Interpret & rank

{Interpret findings; rank by priority/severity where it applies; identify what could be made mechanical and handed to a generator.}

Phase 3 — Manifests

For each opportunity worth acting on, compose a manifest: a self-contained, paste-ready spec a generator (or a human) can execute without further judgment.

Phase N — Output declaration

Write _Claude/reports/{name}.html — flush-and-fill. This analyzer produces no other output.

Recommended report shape

The report's structure is the analyzer author's judgment, but a workable default is: a Summary (what was reviewed, headline counts / priorities), then Findings / Opportunities — each with a priority + title, its Source, a one-paragraph description, and a self-contained Manifest a generator or human can run without further judgment.
