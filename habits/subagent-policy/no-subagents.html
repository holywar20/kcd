no-subagents — Habit

When

a task could be decomposed or parallelized and you're tempted to spawn a subagent (the Task / Agent tool, a workflow fan-out, any unsupervised relay)

Action

do the task yourself in the main session; spawn a subagent only when the user asks for one by name, and even then draft the prompt for them to run rather than dispatching it

Explanation

An autonomously-spawned subagent starts cold — it blows the token budget on standup, never receives the lens context the main session carries, and spins rediscovering what it's supposed to do. The separation of concerns is real but currently costs more than it returns. Drafting-not-dispatching keeps the human in the loop, where the leverage is — they run the agent so they can track its progress and step in on confusion or error.

Rules

  • do it in the main session — no subagents to "separate concerns," parallelize, or offload standup
  • exception is explicit only — a subagent solely when the user asks for one by name
  • when they do want one, draft the prompt and hand it over; don't dispatch
