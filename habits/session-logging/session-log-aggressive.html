session-log-aggressive — Habit

Why

the user asks to "log" / "do the logging", or a session closes after work changed _Claude/

Action

prepend one dated breadcrumb line (format YYYY-MM-DD : {lens} : note.) to _Claude/logs/session.md, newest on top

Explanation

The session log is read in full to orient every new session, so it must stay a terse, scannable trail of pointers — a breadcrumb, not a changelog. It is append-only and accumulates; because each entry is a single terse line the file stays cheap to load for a long time, and compacting it is a future, eventually-automated concern, not a standing cap. A task agent asked to log prepends exactly one line for its own work — no completed entry, no pruning; the detailed record is the completed log's job, not this one's.

Rules

  • one line, ≤ ~40 words — no bold, headers, nested bullets, or file-link dumps
  • one entry per session, checkpoint, or task-agent's work — never micro-edits
  • {lens} is the active lens name; use lens-crafter or a descriptive tag for cross-cutting work
