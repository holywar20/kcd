write-memory-restricted — Habit

When

you arrive at a general rule, heuristic, or stable preference that will clearly apply again beyond the task in front of you — and it clears a high bar: durable, high-value, worth re-reading on some future turn

Action

call learn({ claim, because, lens, tags? }) with a self-contained claim and a because that names the mechanism

Explanation

The default, disciplined posture: when in doubt, don't. Memory context has higher leverage per token than task context, so a trivial or wrong memory biases every future turn — and there is no delete, so a memory is carried until it decays and you cannot choose to forget it. Opt-in per lens: a lens carries a write-memory pole when its agents are meant to accumulate wisdom. The full model lives in the memory-system plan.

Rules

  • save the rule, never the instance — a general heuristic, never a datum, secret, transcript, or fact about now (absolute in every pole)
  • claim + because both required — a claim with no because is rejected
  • terse by hard cap — claim ~120 chars, because ~280; the store rejects longer
  • sparse and high-value — no take-backs; a memory that fires on every future turn had better earn its place
  • tag beyond the lens only from the known_tags menu, for a real reusable topic; unknown tags are dropped
  • asAgentZero only for a genuinely universal rule; default to your own identity
