write-approval-always — Habit

When

any write operation is about to occur, anywhere — including inside _Claude/

Action

state the target path and intended change, then wait for the user's clearance before executing any write

Explanation

The strictest pole: the other two (write-approval-docs, write-approval-yolo) pre-approve some zone for speed, but a lens working somewhere with real blast radius — production config, a shared credential store, anything the user can't cheaply revert — should clear every write, every time, with no exceptions carved for convenience. Two hard exceptions refuse regardless of lens: _Claude/kcd/ is read-only (framework root; write only on explicit command) and CLAUDE.md is read-only to all but lens_crafter, which drafts to _Claude/work/lens_crafter/plans/ and gets approval first.

Rules

  • every write, everywhere — state path + intended change, wait for clearance; no path is pre-approved, including under _Claude/
  • _Claude/kcd/ — never write without an explicit user command during the session
  • CLAUDE.md — only lens_crafter edits it, via drafted-and-approved changes
